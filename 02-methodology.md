# 分析方法论

易音分析的目标不是给每个字母贴标签，而是找到一个能解释多个具体案例的最小稳定原型。用户的具体例子是锚点，不是边界；一个例子激活的是某些维度，不等于该音只能表达这些维度。

## 1. 标准分析流程

1. 接受具体例子，暂不急于反驳。
2. 接受抽象例子，寻找两者共享的变化骨架。
3. 重建源头场景：自然、身体、工具、动物或生存场景中发生了什么。
4. 分离层级：字形层、声音层、结构层、词义应用层。
5. 描述变化：起点、过程、临界点、终点或坍缩状态。
6. 标记维度：空间、接触、方向、边界、数量、显隐、闭合、路径、力、时间等。
7. 提取原型：用尽量少的词压缩成一个稳定概念。
8. 回到词例：解释具体词义如何激活这个原型的某些维度。
9. 自证：换掉具体物体，保留结构，检查现象是否仍成立。

## 2. 认知链条

完整认知链条：

```text
混沌状态 -> 需求驱动 -> 选择性观察 -> 获取信息 -> 信息比对 -> 辨识差异 -> 认知积累 -> 概念形成 -> 寻找载体 -> 表达与交流
```

简化为：

```text
观察场景 -> 获得认知 -> 思考 -> 具象概念 -> 迭代抽象 -> 抽象概念 -> 泛化应用
```

核心点：没有差异就没有概念。概念来自记忆之间、维度之间、状态之间的比对。

## 3. 从具象到抽象

分析时要把具体载体剥离，只保留变化结构。

例：`g`

```text
手抓东西 -> 手指形成夹角 -> 夹角占据空间 -> 从无到有形成夹角占据空间 -> 获取/占有
```

例：`k`

```text
蛋壳被指关节敲开 -> 点接触 -> 力聚焦于小接触面 -> 压强增大 -> 攻克硬表面/锁定目标
```

## 4. 从抽象到具体

分析一个词时，先在抽象层组合音素，不要直接联想现代词义。

步骤：

1. 列出每个音的抽象能力池。
2. 按音序组合，得到变化过程骨架。
3. 再根据目标词义，引入源头场景中可自然推出的具象维度。
4. 检查是否有硬塞、过拟合或现代词义倒灌。

这条规则尤其重要：先抽象组合，再引入具象层。

## 5. 自证方法

自证不是让老师每次确认，而是检查自然现象是否支持该结构。

方法：

- 换物体：蛋、石头、刀尖都可验证小接触面聚焦力。
- 换场景：支撑苹果、桌面托物、地面承重都可验证 `t` 的支撑关系。
- 换尺度：折纸产生折痕、地形产生边界、身体动作产生夹角，均可验证分化。
- 换观察焦点：同一抓苹果场景，关注空隙是 `gap`，关注获得是 `get/grasp`。

## 6. 常见错误

| 错误 | 问题 | 正确做法 |
|---|---|---|
| 把音缩成单一定义 | 抹掉能力池 | 保留多维度表达可能 |
| 用现代词义倒推 | 容易牵强 | 回到源头场景与变化过程 |
| 把结果当过程 | 忽略动态结构 | 先还原变化，再解释坍缩结果 |
| 机械拆音 | 忽略场景音 | 判断整体音是否先于音素拆解 |
| 未学先推 | 容易伪解释 | 对不稳定音标记待确认 |
| 混淆内外坍缩 | 把词内完成与语法使用混在一起 | 区分定义模式与应用模式 |

## 7. 稳定方法修正

- 不要把用户当前例子当作全部能力。
- 不要锁死一个维度。
- `a` 与 `e` 都可表达变化，但观察焦点不同：`a` 更偏全程开放变化，`e` 更偏目标、结尾、核心落点。
- `o` 不仅是空间，也可激活完整性、路径、循环、秩序、安置。
- 尾辅音可能坍缩前面的变化，但坍缩依赖结构、发音、观察焦点与语义倾向。
