# 待确认与研究计划

本页用于防止 wiki 把尚未稳定的内容写成定论。易音需要持续区分“已稳定原型”“局部激活维度”和“待验证猜想”。

## 1. 待独立展开的音

| 音 | 当前状态 | 已知线索 | 下一步 |
|---|---|---|---|
| `e` | 部分稳定 | 目标点、结尾区、核心落点；与 `a` 互补 | 建立独立源头场景与词例链 |
| `u/w` | 部分稳定 | 闭合、接触、隐藏、状态切换；在 cup/up/put 中方向不同 | 明确开->闭、闭->开两类用法 |
| `n` | 部分稳定 | 重复小接触输出、保留、更替、序列、痕迹 | 建立源头场景与 `an/can/run/union` 链 |
| `y` | 部分稳定 | 环绕、整体、周围、环绕式观察 | 以 sky/tyre/year 等构建对照 |
| `l` | 仅 skill 笔记稳定 | 对齐延伸、线、轮廓、结构传播 | 需要源头场景 |
| `m` | 仅 skill 笔记稳定 | 外边界显现、覆盖、隐藏内部、镜像、溢出 | 需要源头场景 |
| `v/f` | 部分对比 | `f` 与 `b` 都向外，`b` 爆发，`f` 持续流动扩散；`v` 为差异显现 | 拆分 f/v 的发音与场景差异 |
| `i` | 仅 skill 笔记稳定 | 单位输出、流动与坍缩成一 | 需要词例验证 |
| `d, z, j, th, ch, ng` | 未展开 | 暂无稳定文档 | 后续逐个建立源头场景 |

## 2. 需要继续澄清的方法问题

- 场景音与拆解音的边界：哪些音先以整体场景音存在，哪些是后期音素化结果。
- 尾辅音坍缩规则：何时尾辅音只是动作模式，何时负责把前序变化收束成结果。
- 现代拼写与古发音关系：当前文档以声音逻辑为核心，后续需要处理历史音变风险。
- 多义词与同音异源：同一现代词形可能汇入多个历史来源，不能直接假设单一路径。
- 词例验证标准：需要形成“通过多少个跨场景例子才认为原型稳定”的准则。

## 3. 优先研究顺序

1. `u/w`：因为它已出现在 `put/cup/cut/up/union` 等多个关键词中。
2. `e`：因为它直接影响 `get/rest/key` 等目标与结果类词。
3. `n`：因为它已在 mem0 中有重要线索，并影响 `an/can/run/union`。
4. `y`：因为它对 `sky/tyre` 等环绕观察类词有解释力。
5. `l/m/f/v`：补足常见辅音能力池。

## 4. 输出文档维护规则

- 稳定内容写入 [[03-phoneme-map|音素规则图谱]] ([音素规则图谱](03-phoneme-map.md))。
- 词例只要仍依赖未稳定音，就在 [[05-example-analyses|例词分析库]] ([例词分析库](05-example-analyses.md)) 中标注“待确认”。
- 新的用户纠正先进入本页，经过多个例子验证后再上升为稳定规则。
- mem0 负责长期记忆；wiki 负责文档化输出。不要把用户私人偏好混入理论正文。
