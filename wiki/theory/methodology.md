# 分析方法论

易音分析的目标是找到能够解释多个具体案例的最小稳定原型。用户给出的具体例子是锚点，不是边界。

## 标准流程

1. 接受具体例子。
2. 接受抽象例子。
3. 找到两者共享的变化骨架。
4. 重建源头场景。
5. 分离字形、声音、结构和应用层。
6. 描述变化的起点、过程、临界点和结果。
7. 标记观察维度。
8. 压缩成最小稳定原型。
9. 用这个原型解释多个词例。
10. 通过自然现象自证。

## 认知链条

```text
混沌状态 -> 需求驱动 -> 选择性观察 -> 获取信息 -> 信息比对 -> 辨识差异 -> 认知积累 -> 概念形成 -> 寻找载体 -> 表达交流
```

简化为：

```text
观察场景 -> 获得认知 -> 思考 -> 具象概念 -> 迭代抽象 -> 抽象概念 -> 泛化应用
```

## 从具象到抽象

例：`g`

```text
手抓东西 -> 手指形成夹角 -> 夹角占据空间 -> 从无到有形成夹角占据空间 -> 获取/占有
```

例：`k/c`

```text
敲蛋壳 -> 点接触 -> 力聚焦于小接触面 -> 压强增大 -> 攻克硬表面/锁定目标
```

## 从抽象到具体

分析词时，不要先拿现代词义硬套音素。推荐路径是：

```text
音素能力池 -> 音序过程 -> 抽象骨架 -> 词义所需维度 -> 具象解释 -> 自证
```

## 常见错误

| 错误 | 问题 | 修正 |
|---|---|---|
| 把音缩成单一定义 | 抹掉能力池 | 保留多维度可能 |
| 现代词义倒推 | 容易牵强 | 回到源头场景 |
| 结果代替过程 | 忽略变化 | 先还原动态过程 |
| 机械拆音 | 忽略场景音 | 判断整体音是否先出现 |
| 未学先推 | 伪解释风险高 | 标记待确认 |
| 混淆内外坍缩 | 语法和词内结构混乱 | 区分定义模式和应用模式 |
