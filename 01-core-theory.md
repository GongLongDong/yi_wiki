# 核心理论

易音把语言理解为“场景经验经过认知抽象后，由声音承载并传递的变化结构”。它反对把词义看成任意标签，也反对从现代词义倒推固定字母义。分析的入口始终是：场景中发生了什么变化，观察者捕获了哪些差异，这些差异如何被抽象为可复用的概念。

## 1. 变化是基本单位

变化是在一个或多个维度上形成差异量的过程。

- 变化已发生：至少一个维度上的差异量不为 0。
- 变化未发生：被观察的全部维度上差异量都为 0。
- 时间不是孤立实体，而是空间变化的一种转换和标记方式。
- 程度、数量、颜色、热量等，也都可以被理解为某种空间关系或微观运动的转换。

易音分析中，词义不是第一层对象。第一层对象是“变化”。物、状态、位置、结果，往往只是变化完成后的坍缩形态。

## 2. 音源场景

每个稳定音都应尽量追溯到一个音源场景。音源场景不是故事装饰，而是音与义发生关联的入口。

基本链条是：

```text
场景发生变化 -> 人观察/经历 -> 捕获差异 -> 形成认知 -> 抽象概念 -> 寻找声音/动作/符号作为载体
```

例如：

- `a` 来自嘴唇从闭到开的发音与出生哭声场景，核心是开放、拉开、空间形成。
- `b` 来自木柴或竹节受热后内部积累、膨胀、裂开或爆炸的场景，核心是内部向外突破。
- `k/c` 来自吃生蛋的 `ko` 场景：球体接触、点接触、力聚焦、攻克硬壳。
- `s` 来自蛇的嘶声、游走、吐舌探测与细长身躯。

## 3. 声音优先，字形次之

易音目前的核心原则是：声音承载变化结构，字形更多是视觉容器。分析时不要把字母形状当成固定意义，也不要因为同一个字母在不同发音中出现，就认为它必然表达同一结构。

这带来两个实践要求：

- 先分析实际声音角色，再看书写形式。
- 不把现代拼写当作原初场景的直接证据。

## 4. 场景音与拆解音

重要修正：`ba` 这类声音可能首先是整体场景音，而不是 `b + a` 机械拼成的组合。`b` 和 `a` 可能是人后来从场景音中拆解出的音素。

因此分析时要避免过早拆解：

- 先问这个整体音是否来自一个强场景。
- 再看其中哪些音素被拆解、稳定、复用。
- 最后才进入组合规则。

## 5. 多维度能力池

一个音不是一个死定义，而是一个表达能力池。源头场景中有多个可观察维度，不同词会激活不同维度。

例如 `b` 的源头场景可以产生：

- 膨胀：bulk, big
- 裂开：break
- 二分/双：bi-, bicycle
- 边界：border
- 相反运动：but, battle, buy
- 爆发：bomb, burst
- 反弹：bump

同一个音在同一个词里也可能同时激活多个维度。表达概念时，维度越多，概念越收敛。
